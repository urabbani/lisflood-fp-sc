## Documentation Conventions

Throughout the documentation, we use the following conventions:

- Code and command-line examples are shown in `monospace font`
- Parameter names are in `monospace font`
- File paths are in `monospace font`
- Optional parameters are indicated with [square brackets]
- Placeholder values that should be replaced are in `<angle brackets>`
